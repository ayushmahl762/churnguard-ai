# Power BI / Tableau Integration Guide

This guide explains how to connect your data visualization tools (Power BI, Tableau, or Excel) to the predictions and analytics generated by the **ChurnGuard AI** pipeline.

---

## 1. Export Data to CSV (Recommended for Power BI & Tableau)

Since Power BI Desktop on Windows does not support SQLite databases natively without third-party drivers, the easiest way to connect your data is to export the database tables and analytical views as CSV files.

We have provided a utility script that automates this. To run it:

1. Open your terminal at the root of the project.
2. Run the following command:
   ```bash
   .venv\Scripts\python.exe dashboard/export_for_bi.py
   ```
3. This will create a `dashboard/exports/csv/` folder containing the following clean CSV datasets:
   * **`customers.csv`**: Demographics and customer profile metrics.
   * **`orders.csv`**: Detailed transactions and interactions.
   * **`monthly_revenue.csv`**: Macro-level historical revenue data.
   * **`product_summary.csv`**: Catalog-level product returns and ratings.
   * **`churn_predictions.csv`**: ML model outputs (Customer ID, Churn Probability, Risk Tier).
   * **`shap_importance.csv`**: Model feature importance importances (frequency score, dormant flags, etc.).
   * **`v_executive_kpis.csv`**: High-level board report KPIs.
   * **`v_churn_by_tier.csv`**: Retention performance by membership tier.
   * **`v_churn_by_channel.csv`**: Marketing channel acquisition performance.
   * **`v_revenue_trend.csv`**: Historical growth trends.

---

## 2. Ingest CSVs into Power BI

### Step 1: Import the Data
1. Open **Power BI Desktop**.
2. Click **Get Data** -> **Text/CSV**.
3. Select and import all files located in `dashboard/exports/csv/`.
4. Click **Load** to import them into the data model.

### Step 2: Establish Model Relationships
Go to the **Model View** (left-sidebar diagram icon) and ensure the tables are linked. Power BI should auto-detect most relationships, but verify the following mappings:

1. **`customers`** to **`orders`**:
   * Field: `customer_id`
   * Relationship: **1 to Many** (One customer has many orders)
   * Cross filter direction: **Both**
2. **`customers`** to **`churn_predictions`**:
   * Field: `customer_id`
   * Relationship: **1 to 1** (One prediction row per customer profile)
   * Cross filter direction: **Both**

## 3. Recommended Visualizations

Use the following layout plan to build a presentation-grade dashboard:

### Page 1: Executive Overview (KPIs & Trends)
* **KPI Cards (Top Row):**
  * **Overall Churn Rate**: `AVERAGE(v_executive_kpis[churn_rate_pct])`
  * **Total Revenue at Risk**: `SUM(churn_predictions[churn_probability] * customers[total_spend_usd])`
  * **Average LTV (Retained vs. Lost)**: Compare average spend using `v_executive_kpis[avg_ltv_usd]`.
* **Revenue Trend (Line Chart):**
  * X-Axis: `v_revenue_trend[period_label]`
  * Y-Axis: `v_revenue_trend[revenue_usd]`
  * Tooltip/Second Line: `v_revenue_trend[unique_customers]`
* **Churn by Tier (Donut Chart):**
  * Legend: `v_churn_by_tier[membership_tier]`
  * Values: `v_churn_by_tier[churned]`

### Page 2: Customer Risk Analysis & Campaign List
* **Risk Tier Distribution (Bar Chart):**
  * X-Axis: `risk_tier` (LOW, MEDIUM, HIGH)
  * Y-Axis: `COUNT(customer_id)`
* **Acquisition Performance (Bar Chart):**
  * X-Axis: `v_churn_by_channel[acquisition_channel]`
  * Y-Axis: `v_churn_by_channel[churn_rate_pct]`
* **Actionable Retention List (Table Visual):**
  * Columns: `customer_id`, `membership_tier`, `total_spend_usd`, `days_since_last_purchase`, `churn_probability`
  * Filter: Add a visual filter to only show rows where `risk_tier = "HIGH"`.
  * Sort: Sort descending by `churn_probability` or `total_spend_usd` to prioritize high-value targets.
